## 为什么写博客

其实已经写过好多次了…… 这应该是我坚持最久的一次，每次的原因都不同：

- 最早，是觉得写博客很酷，也想有一个地方发表自己写在 [Obsidian](https://obsidian.md/) 里面的文章，不想只是烂在电脑里面
- 后来觉得博客可以作为「在线作品集」，只需要贴出链接给需要的人就好啦（虽然完全没有这种需求）
- 现在，是在减少社交媒体的使用后，给自己当下的想法找一个出口

我觉得这个转变有一个很重要的内核，就是我逐渐开始接受自己「自然」的表达，而不会斟酌很久导致完美主义症状发作。毕竟[好的系统都是逐步构建](/TODO)的嘛！

## 我用什么平台管理

一直用的都是 [GitHub Page](https://pages.github.com/) 来部署，你可以在[这里](https://github.com/avarbykira/avarbykira.github.io)看到我的代码仓库。GitHub Page 是完全免费的，但只能部署静态页面，不过反正我只是需要一个网页来展示文章而已。默认的网址是 `[你的 GitHub ID].github.io`，但你也可以改成自己的域名。我的域名是 `www.tcli.me`，`tcli`是我的姓名缩写，`.me` 的一级域名也符合博客的主题，还蛮不错的吧！

博客的前端框架用的是 [Astro](https://astro.build/)，页面布局则用的是开源免费的 [Dante](https://github.com/JustGoodUI/dante-astro-theme) ，我不是很熟悉前端框架，但在 Codex 的帮助下还是完成了一些自定义工作，比如博客首页的「Latest」部分就是 Vibe Coding 出来的。如果你像我一样有强迫症，希望完全自定义博客页面，推荐你选择自建站点，在 AI 的帮助下应该还是蛮容易的。

## 有其他平台写文章吗

很少。通常来说，我不会在朋友圈、小红书这样的平台发长篇大论，毕竟算法不青睐这样的内容，而给现实生活中的朋友看到这种程度的自我披露，也让我觉得有些 cringe. 

我曾在少数派发过[一篇文章](https://sspai.com/post/71545)讨论如何控制玩手机的欲望，还拿了当年的年度征文，不过总的来说，我一般不会这样长篇大论地写文章。

在某段比较 down 的时期，我也在 bilibili 上频繁发过一些非常意识流、没什么逻辑的文字，但最近也不怎么发了。

## 我如何写文章

以前会想用 Obsidian 写文和管理，但看习惯 VSCode 后，感觉还是直接在 IDE 里面写，更不容易因为格式和排版问题分心。

虽然 Wiwi 是命令行的鼓吹者（顺带一提，他的鼓吹真的很有用，在他的影响下我已经改变了很多使用电脑的习惯），但我暂时还不打算做到那么 Geek（笑）

## 我何时有写作灵感

我通常会在每天早上到工位后，趁着早上没有什么活儿把今天的文章写掉。感觉早上的自己脑子最清醒，这样的好时间怎么舍得留给工作 :-P

至于灵感，毕竟博客刚起步嘛，肯定不至于枯竭。我也会在文章里留下一些跳转链接（其实这篇里面就有！），等未来再单独开一篇详细聊一聊。

通常我的灵感会随机产生，遇到了我就记下来，等想起来了再写成完整的文章。

## 会存草稿吗？

通常不会。说来也巧，我昨天才给博客新增了草稿功能，具体来说，是当我在 front matter 里将 `isDraft` 字段设为 `true` 的时候，这篇文章是不会展示在博客上的，只能从后台看见。

写这个功能不是为了攒着慢慢发（虽然也不是不可以），而是难免会遇到文章写一半就下班了的情况，我可以把没写完的文章推到 GitHub 上，然后换个地方继续写。

## 最喜欢的文章是哪一篇？

每一篇 :-D
